# Branch Review — Deterministic Northbridge MVP

This note summarizes what the current feature branch delivers, how to run and
test it, what is deliberately deferred, and the recommended next step before
AI integration is considered.

## What this branch implements

A runnable, deterministic, end-to-end **Northbridge Water Failure** campaign on
a single machine, with no AI and no durable storage.

- **Framework-free deterministic engine** (`engine/`): plain Python dataclasses.
  - 16 state variables, clamped to 0–100.
  - 10 factions with posture labels that update from state.
  - 6 advice options, each with distinct tradeoffs.
  - 10 per-turn client calls (one per playable turn).
  - Ambient crisis drift, NPC decision logic
    (`FOLLOWED`/`PARTIALLY_FOLLOWED`/`MODIFIED`/`DELAYED`/`REJECTED`),
    scaled advice effects, and an `AppliedDiff` for every change.
  - Failure thresholds and 10-turn completion (`ACTIVE`/`COMPLETED`/`FAILED`).
- **FastAPI backend** (`backend/`): Pydantic v2, in-memory campaign store.
  - `GET /health`
  - `POST /api/campaigns`
  - `GET /api/campaigns/{id}`
  - `GET /api/campaigns/{id}/current`
  - `POST /api/campaigns/{id}/advice`
  - `GET /api/campaigns/{id}/turns`
- **React + TypeScript + Vite frontend** (`frontend/`): a diegetic
  crisis-consulting workstation — state panel, client call, advice workbench,
  aftermath with applied diffs, turn history, and a canon archive.
- **Tests** (`tests/`): determinism, 0–100 bounds, failure thresholds,
  completion at turn 10, applied diffs, and engine/FastAPI independence.

The loop is fully playable: start a campaign, read the call, choose advice,
advance, see the NPC decision + applied diffs + aftermath, repeat to turn 10
or failure, and review the turn history.

## How to run it

Prerequisites: Python 3.10+ and Node.js 18+.

Backend (from the repository root):

```bash
python -m venv .venv
# Windows PowerShell
. .venv\Scripts\Activate.ps1
# macOS / Linux
# source .venv/bin/activate

pip install -e "backend[dev]"
uvicorn app.main:app --reload      # http://localhost:8000  (docs at /docs)
```

Frontend (in a second terminal):

```bash
cd frontend
npm install
npm run dev                        # http://localhost:5173 (proxies /api + /health)
```

Start the backend first; the Vite dev server proxies API calls to it.

## How to test it

Engine + invariants (from the repository root, venv active):

```bash
pytest
```

These tests import only `engine/` and require no server. They currently number
35 passing, covering bounds, applied diffs, turn increments, every failure
threshold, 10-turn completion, determinism, and FastAPI-independence.

Frontend typecheck + production build:

```bash
cd frontend
npm run build      # runs tsc -b && vite build
```

Manual end-to-end smoke: with the backend running, create a campaign
(`POST /api/campaigns`), fetch `/current`, then submit one of the six advice
ids repeatedly to `/advice` until `status_after` becomes `COMPLETED` or
`FAILED`. A balanced rotation (e.g. `controlled_disclosure`,
`contractor_pressure`, `mutual_aid`, ...) completes; always choosing
`delay_disclosure` fails on legal exposure.

## What is intentionally missing

- **No AI / model calls.** No provider abstraction, no prompts, no
  `ModelRun` logging, no structured-output validation.
- **No durable persistence.** Campaigns live in process memory and are lost on
  restart. No SQLite/Postgres/SQLAlchemy.
- **No documents / evidence board / memo-drafting workbench.**
- **No dossier export.**
- **No authentication or deployment configuration.**
- **No statewide / regional / interstate progression.** Town-level only.

See `docs/architecture.md` § "Intentionally Not Present Yet" for the full list.

## Recommended next implementation step

The deterministic loop and the workstation UI are already demonstrably usable
end to end, so the next step is **UI polish and richer Northbridge content** —
not AI integration:

1. **Richer scenario content:** introduce the `Document` entity and a first set
   of Northbridge documents (preliminary lab report, town-manager call
   transcript, contractor warning letter, hospital water-priority request,
   resident rumor thread) behind an Evidence Board, all classified facts.
2. **Advice Workbench depth:** let advice carry a short rationale/expected
   benefits & harms surfaced in the UI, and add 2–4 more situation-specific
   advice options per turn so choices feel context-dependent rather than a
   fixed six.
3. **Richer aftermath:** deterministic faction-reaction and media-framing text
   keyed off the resulting state (still generated by rules, not models), and a
   low-fidelity "last verified / degraded feed" treatment as `staff_capacity`
   or `information_integrity` drop.
4. **Dossier export:** serialize turn history + canon to a Markdown campaign
   dossier.

AI integration (a validated, read-only Research Console that only *proposes*
classified facts) should come **after** the above, so the deterministic,
document-rich loop is the stable foundation the model layer is layered onto.
